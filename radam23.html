Case Study.

1. ABOUT
rAdam23
rAdam23 is a personal portfolio website to showcase my current projects and case studies.
2. MY ROLE
Full-Stack Web Designer
  • Front end web development
  • Graphic Design
  • User Interface Design
  • User Experience Testing
3. REQUIREMENTS
The Brief
For the website I needed to showcase my current and upcoming projects across code and design. The requirements of the brief included:
  • User Interface design
  • User Experience design
  • Graphic design,and
  • Coding (HTML/CSS)
Another requirement was that the final site was not over wordy yet retain the polished, lightweight feel and ease of more famous tech sites & brands (see Atlassian, Trello, Twitter, Github) on the market today.

The intention here was for it to act as a brand on its own rather than an extension of a cv as rAdam23 brand grew in online presence overtime.
4. THE WEBSITE
Needs & Pain points
For this project I needed the live website to meet the following criteria:
  • Simple to navigate (fewer clicks is better).
  • Visually elegant (providing a platform for current and future case studies & projects)
    • and
  • Hand-coded (must be lightweight and fast in load times)
  • Mobile responsiveness (desktop, tablet and mobile)
  • Upgradeable (code must be semantic, logically written and clean to enable ease of future upgrades)
5. CREATING
Moodboards
Inspiration for the website was drawn from major tech companies websites & web-based applications.

A chunky Source Sans Pro font, iconography and web-based animations were inspired from the fun & 'techy' feel of major tech companies such as Miro, Github, Figma etc.
Pantone 282C blue was chosen as the primary color for the website along with font Source Sans Pro, presenting a cool, calm & clean look.

6. DESIGNING
User Flows
Userflows for the website were created and simplified to ensure the site demanded as minimal effort as possible from the users to navigate and view content.

Initial aim of the userflows for rAdam23 were to reduce the amount of clicks and travel from the users. This increased accessibility to key areas, reduced user fatigue as time to acces these areas (namely case studies) were speedy and effortless.
The reduction of steps in the userflows and a careful focus on design/content within the site had the added benefit of reducing load times significantly as only essential content was included.

7. CODING
The Front End
Coding the site was a lot of trial and error with hundreds of hours spent understanding HTML/CSS technologies and building. Github acted as a public journal updates & site evolution from inception.

Throughout the process of developing the site and as I became more familiar in the structure,

I refactored the code.. a lot.
So much so that first-attempt versions of the static site contained the css contained 3,000 lines of code alone. As I refactored and refined the site, code reduced by 1/3 while visually becoming more appealing.

Javascript animations to improve user experience were left until last when I was satisfied I had most efficient version of the code.

The Website
